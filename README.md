# Multi-Page Automation

一个用于自动完成 ChatGPT OAuth 注册与授权流程的 Chrome 扩展。

适用场景：
- 批量跑注册与授权流程
- 通过邮箱自动收取验证码
- 将最终授权结果提交到 CPA 面板，或直接在本地下载结果文件

## 功能特性

- 9 步自动化流程：获取 OAuth、注册、验证码、授权、回调处理
- 支持批量自动运行，可配置执行轮数
- 支持 `163`、`QQ`、`2925` 三种邮箱模式
- `2925` 模式支持“邮箱前缀 + 随机后缀”自动生成邮箱
- 步骤 4、7 自动轮询邮箱，未找到验证码时会自动触发重发并继续重试
- 步骤异常支持人工介入后继续执行
- 未填写 CPA 接口地址时，最终结果可直接下载为本地 JSON 文件
- 步骤 8 遇到 OAuth 授权页异常时可快速失败，自动模式下会重开当前轮

## 运行要求

- Chrome 浏览器
- 已开启开发者模式
- Chrome 窗口在运行过程中必须保持可见，不能最小化
- 对应邮箱站点必须提前登录

说明：

本扩展依赖真实页面 DOM、焦点和点击事件。窗口被最小化或长时间切到不可交互状态时，自动化点击和输入可能失效。

## 支持的邮箱模式

### 163 / QQ

适合配合 DuckDuckGo Email Protection 使用。

准备方式：
- 准备一个 `@duck.com` 邮箱地址
- 将 DuckDuckGo 转发目标设置为 163 或 QQ 邮箱
- 在插件中粘贴 `@duck.com` 地址，验证码由 163 / QQ 邮箱页面自动读取

### 2925

准备方式：
- 在侧边栏选择 `2925 邮箱`
- 填写邮箱前缀，例如 `abc`
- 步骤 3 会自动生成邮箱地址，例如 `abcx8d2kp@2925.com`

## 安装方式

1. 打开 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择当前项目目录
5. 打开扩展侧边栏 `Multi-Page Automation`

## 侧边栏配置说明

| 字段 | 说明 |
| --- | --- |
| CPA 接口 | VPS / CPA 面板地址。留空则最后走本地下载模式 |
| 邮箱 | 选择 `163` / `QQ` / `2925` |
| 邮箱前缀 | 仅 `2925` 模式显示 |
| 邮箱地址 | `163/QQ` 模式下手动填写；`2925` 模式下自动生成 |
| OAuth | 第 1 步完成后自动显示 |
| 回调 | 第 8 步完成后自动显示 |

## 使用流程

### 自动模式

点击侧边栏中的“自动”后，插件会按以下顺序执行：

1. 获取 OAuth 链接
2. 打开注册页
3. 填写邮箱 / 密码
4. 获取注册验证码
5. 填写姓名 / 生日
6. OAuth 登录
7. 获取登录验证码
8. 完成 OAuth 授权
9. CPA 接口验证 / 本地下载

### 第 3 步差异

- `163 / QQ` 模式：流程会暂停，等待手动粘贴邮箱地址后继续
- `2925` 模式：不会暂停，插件自动生成邮箱并继续

### 第 4 步与第 7 步

验证码处理逻辑：
- 自动轮询邮箱页面
- 自动识别验证码邮件
- 未找到验证码时自动点击“重新发送”
- 重新进入下一轮轮询

其中：
- 第 4 步读取注册验证码
- 第 7 步读取登录验证码

## 第 8 步说明

第 8 步会在 OAuth 同意页点击“继续”，并监听跳转到本地回调地址。

如果遇到以下异常页面：
- `invalid_state`
- “糟糕，出错了”
- “验证过程中出错”

则会判定当前 OAuth 会话无效。

当前行为：
- 手动模式：步骤直接失败，便于你重新开始
- 自动模式：当前轮会直接作废并重开，避免卡死在错误页面

## 第 9 步说明

### 填写了 CPA 接口地址

插件会：
- 打开 CPA / VPS 面板
- 将回调 URL 填入面板
- 提交验证

### 未填写 CPA 接口地址

插件会：
- 在本地根据回调参数完成 token 处理
- 自动下载 JSON 结果文件到浏览器默认下载目录

## 人工介入机制

如果某些步骤无法自动完成，插件会进入人工介入模式，并在侧边栏显示提示。

此时你可以：
- 手动处理当前页面
- 点击“人工处理完成，继续下一步”

适用场景包括：
- 页面结构变化
- 特殊风控页面
- 某一步按钮或输入框未被正确识别

## 输出结果

当走本地下载模式时，会自动下载一个 JSON 文件。

结果中通常包含：
- `access_token`
- `refresh_token`
- `id_token`
- `account_id`
- `email`
- `expired`
- `last_refresh`
- `type`

## 注意事项

- 运行期间不要手动频繁切换标签页
- 不要最小化 Chrome
- 邮箱页面必须保持登录状态
- 若页面结构发生变化，相关步骤可能需要更新选择器
- 自动批量模式下，步骤 8 的 OAuth 错误不会无限重试，当前版本设置了单轮最多重开 2 次

## 开发说明

- `background.js` 负责步骤调度、状态管理、自动运行和标签页控制
- `content/signup-page.js` 负责 OpenAI 注册、登录、验证码和授权页交互
- `content/vps-panel.js` 负责 CPA / VPS 面板交互
- `content/mail-163.js`、`content/qq-mail.js`、`content/mail-2925.js` 负责邮箱轮询
- `sidepanel/` 目录负责侧边栏 UI 和交互逻辑
